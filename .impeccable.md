## Design Context

### Users

Mixed audience across three tiers:
- **Solo devs / self-hosters** — technical, high ownership, often power users exploring every feature
- **Small team admins** — managing API keys and billing for a team; care about clarity and control
- **Platform operators** — running shared AI services for downstream users; need at-a-glance health and dense data

All three use this at a desk, in a browser, during focused work sessions. Not casual. Not on mobile. They trust the tool to be correct, fast, and predictable — not exciting.

### Brand Personality

**3 words: calm, precise, trustworthy**

The interface should feel like a well-maintained instrument panel: no unnecessary decoration, everything exactly where you expect it, confidence-inspiring at a glance. Think high-quality technical documentation printed on good paper. Not cold. Not aggressive. Not trying to impress — just doing its job with obvious care.

Avoid: startup energy, "delight," rounded-everything-with-gradients, dashboard glamour.

### Aesthetic Direction

**Reference:** Linear, Vercel Dashboard — refined SaaS admin aesthetics, sharp type hierarchy, tight spacing, everything earns its place.

**Theme:** Light mode (suits the "precise, trustworthy" register; dark mode reads as drama, which conflicts with the brand tone).

**Color:** Extend the existing sage green palette into OKLCH. The green (#3c8a61 ≈ oklch(52% 0.11 148deg)) is the right instinct — it signals "go," "health," "stable." Don't abandon it. Instead, tint all neutrals toward it at minimal chroma (0.005–0.01) for subconscious cohesion. Avoid cyan/teal drift. Keep backgrounds warm-green, not cool-gray.

**Typography:** Replace Inter with a distinctive pairing:
- **Body:** Figtree — humanist sans, warm but precise, excellent x-height for UI density, not trendy
- **Display/Headings:** Epilogue — more geometric character, slight editorial weight contrast with Figtree
- **Monospace:** Keep JetBrains Mono — it's correct for code/tokens

### Design Principles

1. **Precision over decoration** — every element must justify its presence. If removing it doesn't break comprehension, remove it.
2. **Hierarchy through contrast** — use at most 5 type sizes with at least 1.25× ratio between steps. Space above headings, not below.
3. **Green means action** — the sage accent is reserved for interactive states, indicators of health/progress, and CTAs. Don't dilute it with decorative use.
4. **Data first, chrome last** — admin consoles are read, not admired. Padding, borders, and UI chrome should recede; content should dominate.
5. **Consistency earns trust** — spacing scale, border radius, shadow depth, and color use must be utterly consistent. One oddball component breaks the mental model.
